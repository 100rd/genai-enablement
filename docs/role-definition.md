# GenAI Adoption Leader - DevOps/SRE Specialist

## Role Summary

The GenAI Adoption Leader (DevOps/SRE) is a specialist within the GenAI Adoption Team (GenAI Value Lab). This role designs how GenAI fits into CI/CD, infrastructure, observability, incident response, and reliability workflows, then helps project teams adopt and own it themselves.

## Core Responsibilities

### 1. GenAI in DevOps & SRE Workflows
Design GenAI-augmented workflows for:
- **CI/CD**: Pipeline authoring, YAML generation, policy-as-code assistance
- **Infrastructure-as-Code**: Terraform/Kubernetes manifest refactoring and reviews
- **Log & Metric Analysis**: Incident triage, post-mortem generation
- **Runbook Creation**: Knowledge search and automation for on-call

Turn ad-hoc "ask the AI about this error" into repeatable practices and tools.

### 2. Tooling, Platforms & Pipelines
- Prototype and refine utilities, prompts, and scripts
- Integrate with common stacks: GitHub Actions, GitLab CI, Jenkins, Kubernetes, Helm, Terraform, Argo CD
- Evaluate platform vs custom approaches
- Implement guardrails for safe production use

### 3. Reliability & SRE Practices
Embed GenAI into SRE routines:
- SLO/SLI definition support
- Incident summarization and timeline extraction
- Remediation suggestions based on historical incidents
- Reduce MTTR, alert noise, and toil without hiding risk

### 4. Enable, Don't Own
- Run workshops, live demos, and pairing sessions
- Move teams from "AI as a side toy" to documented, auditable, safe usage
- Capture patterns as playbooks, templates, and case studies
- Knowledge transfer for long-term team ownership

### 5. Measure Impact
Track DORA metrics and human impact (see `success-metrics.md`).

## Required Skills
- Deep experience with CI/CD pipelines and IaC (Terraform, Kubernetes)
- Strong understanding of observability and incident management
- Familiarity with LLM capabilities and limitations
- Ability to prototype quickly (Python, shell scripting)
- Communication skills for workshops and enablement

## Engagement Model

```
Assess (1-2 weeks) -> Pilot (2-4 weeks) -> Scale (4-8 weeks) -> Handoff
```

Each engagement produces:
1. Assessment report with baseline metrics
2. Customized tool stack recommendation
3. Implemented AI-assisted workflows
4. Playbook for the team to own going forward
5. Impact report with measured improvements
