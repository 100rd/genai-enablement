# terragrunt-skill — Local Delta (CLI redesign + backend changes)

The vendored `terragrunt-skill` (jfr992, pushed 2026-04-30) is **current on structure**
(`root.hcl`, Stacks / `terragrunt.stack.hcl`, units, `values.xxx`, catalog/live, nested
`.terragrunt-stack/`). This delta records **command + backend** changes through
**Terragrunt 1.0.x** (1.0.0 = 2026-03-30, first release with a 1.x backwards-compat
guarantee) that the skill's references don't fully reflect. Apply on top of `SKILL.md`.

## CLI redesign — old commands REMOVED, use the new ones

| Old (removed/deprecated) | New | Removed in |
|---|---|---|
| `run-all <cmd>`, `plan-all`, `apply-all`, `destroy-all`, `output-all`, `validate-all` | `terragrunt run --all <cmd>` | **v0.84.0** (2025-08-01) |
| `hclfmt` | **`terragrunt hcl fmt`** | CLI redesign |
| `hclvalidate`, `validate-inputs` | `hcl validate`, `hcl validate --inputs` | CLI redesign |
| `render-json` | `render --json -w` | CLI redesign |
| `graph-dependencies` | `dag graph` | CLI redesign |
| `terragrunt-info` | `info print` | v0.90.0 (2025-10-10) |
| `output-module-groups` | `find --dag --json` / `list --dag --tree` | CLI redesign |
| 7 legacy targeting flags | single **`--filter`** query system (+ `--queue-*`) | 1.0 |

New commands worth using: `run`, `run --all`, `exec`, `find` (JSON config discovery),
`list`, `backend bootstrap|migrate|delete`, `dag graph`, `info print`, `hcl fmt`, `render`.

> ⚠️ The vendored skill's `references/multi-account.md` still shows `terragrunt run-all apply`
> — treat that as **`terragrunt run --all apply`**.

## Backend — provisioning is now OPT-IN (the "backend settings" change)

- **v0.87.0 (2025-09-05): automatic backend provisioning removed.** Terragrunt no longer
  silently creates the S3 bucket / lock resource — it **errors** if bootstrap is needed and
  not requested.
- Opt in with **`--backend-bootstrap`** or **`TG_BACKEND_BOOTSTRAP=true`**, or run it
  explicitly: **`terragrunt backend bootstrap`**.
- New dedicated commands: `backend bootstrap` (provision), `backend migrate` (move state
  between backends), `backend delete` (remove state).
- `remote_state` / `generate` block *shape* is unchanged — only the provisioning trigger
  became explicit. Pair with OpenTofu/Terraform native `use_lockfile` (no DynamoDB needed).

## Env vars & flags
- Env prefix is **`TG_`** (not `TERRAGRUNT_`); e.g. `TG_PROVIDER_CACHE`, `TG_BACKEND_BOOTSTRAP`.
- All `--terragrunt-*` flags removed (v0.85.0); drop the `terragrunt-` prefix.
- **v0.88.0**: unknown commands are no longer auto-forwarded to OpenTofu — use
  `terragrunt run -- <cmd>` for pass-through.

## Authoring changes
- **v0.91.0 (2025-10): inputs are no longer auto-read from dependencies.** Wire cross-unit
  data explicitly via `dependency.<name>.outputs.*` (with `mock_outputs` for plan-time).
- Generated `.terragrunt-stack/` directories must be **git-ignored** (regenerated by
  `terragrunt stack generate`).
- Docs moved to **docs.terragrunt.com** (old `terragrunt.gruntwork.io` 308-redirects).

## Net rule for writing/validating Terragrunt today
Verification loop: `terragrunt hcl fmt` → `terragrunt run --all validate` → `tflint`/`checkov`
on the underlying modules → `terragrunt run --all plan`. Backend first-run/CI needs
`--backend-bootstrap`. Never emit `*-all`, `hclfmt`, `render-json`, `graph-dependencies`.

Sources: gruntwork-io/terragrunt releases + docs.terragrunt.com + issue #3535 (deprecation schedule) + Terragrunt 1.0 blog, verified 2026-06-11.
